# FORECAST OS — GitHub Pages Deploy Guide
## You'll be live in under 10 minutes. Promise.

---

## STEP 1 — Create a GitHub account (if you don't have one)
Go to: https://github.com
Sign up. Free. Takes 2 minutes.

---

## STEP 2 — Create a new repository
1. Click the green "New" button (top left)
2. Repository name: `forecast-os`
   (This becomes your URL: yourusername.github.io/forecast-os)
3. Set to: PUBLIC
4. Click "Create repository"

---

## STEP 3 — Upload your files
1. Click "uploading an existing file" (you'll see this link on the empty repo page)
2. Drag ALL of these files into the upload area:
   - index.html
   - nq.html
   - es.html
   - mcl.html
   - gc.html
   - si.html
3. Scroll down → click "Commit changes"
4. Wait 30 seconds

---

## STEP 4 — Turn on GitHub Pages
1. Click "Settings" tab (top of your repo)
2. Click "Pages" in the left sidebar
3. Under "Branch" → select "main" → click Save
4. Wait 60 seconds
5. Refresh the page — you'll see your live URL

---

## STEP 5 — Point Restream at your terminal
1. Go to: https://restream.io (free account)
2. Connect YouTube + Twitch + X
3. In Restream → "Browser Source" → paste your NQ terminal URL
4. Set resolution: 1920x1080
5. Hit Go Live
6. Done. Stream runs 24/7 on Restream's servers.

## UPDATING YOUR TERMINALS (when we add real data, new features, etc.)
1. Go to your GitHub repo
2. Click the file you want to update
3. Click the pencil icon (Edit)
4. Paste new code
5. Click "Commit changes"
6. Live in 60 seconds. No server restart. No RDP. Nothing.
